## 概要
GitHubリポジトリ検索・閲覧アプリ（Android）の実装タスク一覧。  

※ タスクが完了したものについては、「✅️」にチェックを入れています。

## 必須タスク
- [x] なぜその設計を選んだのか」「なぜそのライブラリを使ったのか」といった背景・経緯をドキュメントに記載する
- [x] リポジトリ検索 API から取得したデータを元に、リポジトリの一覧を表示する
- [x] キーワード検索（検索クエリ）が出来る
- [x] ページネーションが実装されている
- [x] 一覧の結果をタップすることで、リポジトリ詳細画面へ遷移する
- [x] なんらかのリポジトリ情報（例: Issue数, Watcher数, ライセンス, README.md の表示など）

## 加点・拡張タスク
- [x] アプリが大規模になっていくことを見越した設計パターンの導入 (MVVM, Clean Architecture, TCA など)
- [x] チームが大規模になっていくことを見越したツールの導入 (SwiftLint, SwiftFormat, Danger など)
- [x] 検索体験の向上（検索実行のタイミング制御 (デバウンス)、ローディング・Empty・エラー状態のUI表現など）
- [x] パフォーマンスの最適化（リスト表示の最適化、画像のキャッシュ戦略など）
- [x] チーム開発を意識した git および GitHub のアクティビティ (Conventional Commits, Pull Request の体裁など)
- [ ] ユニットテスト、（可能であれば）UIテストの導入
- [x] ブックマーク機能の実装 (一覧と詳細での状態共有、ローカルDBへの保存など)
- [ ] アクセシビリティ (Dynamic Type, VoiceOver など) の考慮


## その他
- [x] Personal Access Token を直接ソースに書かないこと
- [x] AI ツール使用箇所は README に記載すること（どの部分で・なぜ使用したか）
